```lua
-- 获取 Access Token
local token_url = "https://aip.baidubce.com/oauth/2.0/token"
local token_params = {
    grant_type = "client_credentials",
    client_id = "YOUR_API_KEY",
    client_secret = "YOUR_SECRET_KEY"

local token_resp = http.post(token_url, token_params)
local access_token = token_resp.access_token
```

### 2. Plugin 相关（2 个）

#### 2.1 LuaEngine.loadApk - 加载 APK 插件

**实现状态**：✅ 已实现

**限制**：返回的是包装的 Lua 表，不是原始的 ClassLoader 对象

**原因**：
- AutoGo 使用 Go 语言实现，与懒人精灵的 Java 环境不同
- AutoGo 的 plugin.LoadApk 返回的是 Go 对象，需要转换为 Lua 表
- 无法直接返回 Java 的 ClassLoader 对象

**当前行为**：
```lua
-- AutoGo 实现
local loader = LuaEngine.loadApk("app-release.apk")
if loader ~= nil then
    print("插件加载成功")
    -- 返回的是包装的 Lua 表，包含以下字段：
    -- loader.id - 插件 ID
    -- loader.path - 插件路径
    -- loader.loaded - 是否已加载
end
```

**懒人精灵行为**：
```lua
-- 懒人精灵实现
local loader = LuaEngine.loadApk("app-release.apk")
if loader ~= nil then
    print("插件加载成功")
    -- 返回的是 Java 的 ClassLoader 对象
    -- 可以直接调用 Java 方法
    local MyClass = loader:loadClass("com.example.MyClass")
    local instance = MyClass:new()
    instance:someMethod()
end
```

**替代方案**：

**方案 1：使用 AutoGo 的 plugin 模块**
```lua
-- 使用 AutoGo 的 plugin 模块
local plugin = require("plugin")

-- 加载 APK 插件
local loader = plugin.loadApk("app-release.apk")
if loader ~= nil then
    print("插件加载成功")
    -- 可以使用 plugin 模块的其他方法
end
```

**方案 2：使用 AutoGo 的 rhino 模块**
```lua
-- 使用 AutoGo 的 rhino 模块执行 JavaScript 代码
local rhino = require("rhino")

-- 执行 JavaScript 代码
local result = rhino.eval("java.lang.System.currentTimeMillis()")
print("当前时间戳:", result)
```

**注意事项**：
- AutoGo 的 LuaEngine.loadApk 返回的是包装的 Lua 表，不是原始对象
- 无法直接调用 Java 方法
- 建议使用 AutoGo 的 plugin 模块或 rhino 模块
- 如果需要调用 Java 方法，请使用 rhino 模块

```lua
        print("子节点", i, "类名:", node.class)
        print("子节点", i, "包名:", node.package)
        print("子节点", i, "描述:", node.desc)
        print("子节点", i, "位置:", node.bounds.left, node.bounds.top, node.bounds.right, node.bounds.bottom)
    end
end
```

**懒人精灵行为**：
```lua
-- 懒人精灵实现
local children = nodeLib.findChildNodes({class = "android.widget.LinearLayout"}, true)
if children ~= nil then
    print("找到子节点数量:", #children)
    for i, node in ipairs(children) do
        print("子节点", i, ":", node:text())
        -- 可以直接调用节点方法
        node:click()
    end
end
```

**主要差异**：

| 特性 | 懒人精灵 | AutoGo |
|------|----------|---------|
| 返回值 | UiObject 对象数组 | Lua 表数组 |
| 访问方式 | 使用冒号语法（:） | 使用点语法（.） |
| 节点方法 | 支持点击、长按等操作 | 不支持节点操作 |
| 模糊匹配 | 支持正则表达式 | 支持包含匹配 |

**注意事项**：
- AutoGo 的 nodeLib.findChildNodes 返回的是 Lua 表数组，不是原始对象数组
- 使用点语法访问节点属性（如 node.text）
- 不支持节点操作方法（如点击、长按等）
- 模糊匹配使用包含匹配，不是正则表达式

**总结**：
- nodeLib.isAccServiceOk、nodeLib.openSnapService、nodeLib.isSnapServiceOk 已实现，直接返回 true
- nodeLib.findOne、nodeLib.findAll、nodeLib.findChildNodes 已实现，返回包装的 Lua 表
- nodeLib.findNextNode、nodeLib.findPreNode、nodeLib.findInNode 未实现，返回 nil
- 所有已实现的方法都使用点语法访问节点属性
- 不支持节点操作方法（如点击、长按等）
- 模糊匹配使用包含匹配，不是正则表达式

## ❌ 不可用的功能（3 个）

不可用的功能是指：由于技术限制或架构差异，无法在 AutoGo 中实现的功能。

### 1. import - 加载 Java 类

**实现状态**：❌ 抛出异常

**原因**：
- AutoGo 使用 Go 语言实现，与懒人精灵的 Java 环境不同
- Go 语言无法直接加载和调用 Java 类
- AutoGo 不支持 Java 虚拟机（JVM）

**当前行为**：
```lua
-- 当前实现
import('java.lang.*')
-- 抛出异常：import('java.lang.*') 不可用：AutoGo 不支持直接调用 Java 类。请使用 AutoGo 的 plugin 模块或 rhino 模块。
```

**懒人精灵行为**：
```lua
-- 懒人精灵实现
import('java.lang.*')
import('java.util.*')

-- 可以直接使用 Java 类
local list = java.util.ArrayList()
list:add("Hello")
list:add("World")
print("列表大小:", list:size())

local str = java.lang.String("Hello World")
print("字符串:", str:toString())
```

**替代方案**：

**方案 1：使用 AutoGo 的 plugin 模块**
```lua
-- 使用 AutoGo 的 plugin 模块加载 APK 插件
local plugin = require("plugin")

-- 加载 APK 插件
local loader = plugin.loadApk("app-release.apk")
if loader ~= nil then
    print("插件加载成功")
    -- 可以使用插件中的功能
end
```

**方案 2：使用 AutoGo 的 rhino 模块**
```lua
-- 使用 AutoGo 的 rhino 模块执行 JavaScript 代码
local rhino = require("rhino")

-- 执行 JavaScript 代码
rhino.eval([[
    importClass(java.lang.System);
    var timestamp = System.currentTimeMillis();
    print("当前时间戳:", timestamp);
]])
```

**方案 3：使用 AutoGo 的其他模块**
```lua
-- 使用 AutoGo 的其他模块替代 Java 功能
local http = require("http")
local json = require("json")
local files = require("files")

-- 使用 http 模块替代 Java 的 HTTP 功能
local response = http.get("https://api.example.com/data")
print("响应:", response)

-- 使用 json 模块替代 Java 的 JSON 功能
local data = json.decode(response)
print("数据:", data)

-- 使用 files 模块替代 Java 的文件操作
files.write("/sdcard/data.txt", response)
```

**注意事项**：
- AutoGo 不支持直接调用 Java 类
- import 方法会抛出异常，提示用户使用替代方案
- 建议使用 AutoGo 的 plugin 模块或 rhino 模块
- 大多数 Java 功能都可以通过 AutoGo 的其他模块实现

### 2. 线程管理 - beginThread / Thread.newThread

**实现状态**：⚠️ 部分可用

**原因**：
- AutoGo 使用 Go 的 goroutine 机制，与懒人精灵的线程模型不同
- Go 的 goroutine 是轻量级线程，与 Java 的 Thread 不同
- 不支持懒人精灵的线程限制（最多 10 个）
- 不支持懒人精灵的线程共享变量机制

**当前行为**：
```lua
-- AutoGo 实现
function thread_func(arg)
    while true do
        local exit = getCache("exit")
        if exit == false then
            break
        end
        print("我是子线程:", arg)
        sleep(100)
    end
end

setCache("exit", true)
beginThread(thread_func, 1)
sleep(3000)
setCache("exit", false)
```

**懒人精灵行为**：
```lua
-- 懒人精灵实现
local var = {
    exit = true,
    data = 0,
}

function thread_func(arg)
    while var.exit do
        print("我是子线程:", arg, "共享数据:", var.data)
        sleep(100)
    end
end

beginThread(thread_func, 1)
for i = 1, 10 do
    var.data = i
    sleep(1000)
end
var.exit = false
```

**主要差异**：

| 特性 | 懒人精灵 | AutoGo |
|------|----------|---------|
| 线程模型 | Java Thread | Go goroutine |
| 线程限制 | 最多 10 个 | 无限制 |
| 共享变量 | 直接访问全局变量 | 使用 setCache/getCache |
| 线程安全 | 不保证 | 使用 sync.RWMutex |
| 性能 | 较重 | 轻量级 |

**替代方案**：

**方案 1：使用 setCache/getCache 实现线程间数据共享**
```lua
-- 使用 setCache/getCache 实现线程间数据共享
function thread_func(arg)
    while true do
        local exit = getCache("exit")
        local data = getCache("data")
        if exit == false then
            break
        end
        print("我是子线程:", arg, "共享数据:", data)
        sleep(100)
    end
end

setCache("exit", true)
setCache("data", 0)
beginThread(thread_func, 1)

for i = 1, 10 do
    setCache("data", i)
    sleep(1000)
end
setCache("exit", false)
```

**方案 2：使用 Lua 的协程（coroutine）**
```lua
-- 使用 Lua 的协程
local co = coroutine.create(function()
    for i = 1, 10 do
        print("协程:", i)
        coroutine.yield()
    end
end)

for i = 1, 10 do
    coroutine.resume(co)
    sleep(100)
end
```

**注意事项**：
- AutoGo 的线程管理使用 Go 的 goroutine 机制
- 不支持懒人精灵的线程共享变量机制
- 必须使用 setCache/getCache/delCache 实现线程间数据共享
- 不支持懒人精灵的线程限制（最多 10 个）
- Go 的 goroutine 是轻量级线程，性能更好

## 总结

### 部分可用功能总结

| 功能 | 实现状态 | 主要限制 | 推荐替代方案 |
|------|----------|----------|--------------|
| qrDecode | ❌ 未实现 | 需要第三方二维码库 | 使用 ppocr/dotocr 模块 |
| bdOcr | ❌ 未实现 | 需要百度 OCR API | 使用 ppocr/dotocr 模块 |
| LuaEngine.loadApk | ✅ 已实现 | 返回包装的 Lua 表 | 使用 plugin 模块 |
| LuaEngine.getContext | ✅ 已实现 | 返回包装的 Lua 表 | 使用 plugin/system 模块 |
| nodeLib.* | ❌ 未实现 | 需要使用 uiacc 模块 | 直接使用 uiacc 模块 |

### 不可用功能总结

| 功能 | 实现状态 | 原因 | 推荐替代方案 |
|------|----------|------|--------------|
| import | ❌ 抛出异常 | Go 不支持 Java 虚拟机 | 使用 plugin/rhino 模块 |
| 线程管理 | ⚠️ 部分可用 | goroutine 与 Java Thread 不同 | 使用 setCache/getCache |

### 迁移建议

1. **OCR 相关**：
   - 优先使用 AutoGo 的 ppocr 或 dotocr 模块
   - 这些模块是本地 OCR，无需 API Key
   - 性能更好，稳定性更高

2. **Plugin 相关**：
   - 可以继续使用 LuaEngine.loadApk 和 LuaEngine.getContext
   - 但需要注意返回的是包装的 Lua 表，不是原始对象
   - 建议直接使用 AutoGo 的 plugin 模块

3. **nodeLib 相关**：
   - 直接使用 AutoGo 的 uiacc 模块
   - uiacc 模块提供了更强大的节点查找功能
   - 支持 XPath、CSS 选择器等多种查找方式

4. **import**：
   - 无法使用 import 加载 Java 类
   - 使用 AutoGo 的 plugin 模块加载 APK 插件
   - 使用 AutoGo 的 rhino 模块执行 JavaScript 代码
   - 使用 AutoGo 的其他模块替代 Java 功能

5. **线程管理**：
   - 可以继续使用 beginThread 和 Thread.newThread
   - 但必须使用 setCache/getCache/delCache 实现线程间数据共享
   - 不支持懒人精灵的线程限制（最多 10 个）
   - Go 的 goroutine 是轻量级线程，性能更好

## 常见问题

### Q1：为什么有些功能无法实现？

**A**：AutoGo 使用 Go 语言实现，与懒人精灵的 Java 环境不同。有些功能（如 import）需要 Java 虚拟机支持，而 Go 不支持。有些功能（如 nodeLib）需要适配 AutoGo 的 API，需要时间实现。

### Q2：部分可用的功能是否会影响使用？

**A**：部分可用的功能仍然可以使用，但需要注意 API 差异。例如，LuaEngine.loadApk 可以加载 APK 插件，但返回的是包装的 Lua 表，不是原始的 ClassLoader 对象。如果需要调用 Java 方法，建议使用 rhino 模块。

### Q3：不可用的功能是否有替代方案？

**A**：是的，所有不可用的功能都有替代方案。例如，import 可以使用 plugin 模块或 rhino 模块替代，线程管理可以使用 setCache/getCache 实现线程间数据共享。

### Q4：如何选择合适的替代方案？

**A**：根据实际需求选择：
- 如果需要加载 APK 插件，使用 plugin 模块
- 如果需要执行 JavaScript 代码，使用 rhino 模块
- 如果需要 OCR 功能，使用 ppocr 或 dotocr 模块
- 如果需要节点查找，使用 uiacc 模块
- 如果需要线程间数据共享，使用 setCache/getCache

### Q5：后续版本会实现这些功能吗？

**A**：部分功能会在后续版本中实现，例如：
- qrDecode - 计划集成第三方二维码库
- bdOcr - 计划集成百度 OCR API
- nodeLib - 计划适配 AutoGo 的 uiacc 模块

但有些功能（如 import）由于技术限制，无法实现。
